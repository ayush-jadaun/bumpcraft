# Changelog

All notable changes to Bumpcraft are documented here.

This changelog is generated by Bumpcraft itself.

---

## 1.0.3 (2026-03-18)

First public release.

### Features

- Automatic version bumps from Conventional Commits (`feat:`, `fix:`, `BREAKING CHANGE`)
- Pluggable 5-stage pipeline (parse → resolve → changelog → release → notify)
- CLI with 8 commands: release, validate, version, changelog, history, group, plugins, init
- REST API with 7 endpoint groups (health, version, changelog, release, plugins, history, dashboard)
- Programmatic library API (`runRelease`, `currentVersion`, `runReleaseWithCommits`)
- Web dashboard with release preview and one-click release
- Markdown and JSON changelog generation
- GitHub release creation via API
- Release groups — batch commits into named releases
- Release policies — approval gates, freeze windows, daily release caps
- Queryable release history with filters (breaking, scope, type, version range)
- Branch strategies with automatic pre-release tags
- Docker deployment with non-root user, health checks, and nginx example
- Interactive mode with changelog editing via `$EDITOR`

### Security

- Timing-safe API key comparison
- Group name validation against path traversal
- GitHub repo format validation against SSRF
- Non-root Docker container
- Input validation on all API endpoints

### Technical

- TypeScript ESM with NodeNext module resolution
- Zod config validation with format-level checks
- Atomic file writes (tmp + rename) for package.json and history
- Serialized history writes via promise chain mutex
- 77 tests (unit + integration)
- 7 rounds of code audits with all issues resolved
